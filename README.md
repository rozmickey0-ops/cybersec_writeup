# Cybersecurity Lab Writeups

**Date:** 2025-10-05  
**Environment:** TryHackMe   
**Note:** All tasks were performed in an authorized lab environment. Unauthorized testing on real systems is illegal.

---

### Task 1: Identify Security Domain

**Question:**  
Which area of security is involved when finding vulnerabilities?

**Answer:**  
Offensive Security

**What I Learned:**  
Offensive security is about finding and exploiting weaknesses to improve defense. It includes tasks like vulnerability assessment and penetration testing in authorized environments.

---

### Task 2: Identify Fake Data on a Website

**Task:**  
Given access to a practice site `fakebank.com`, locate a fake bank account number.

**Method:**  
1. Opened the provided lab URL.  
2. Used the "View" button to access account details page.  
3. Identified the test/fake account number displayed on the page.

**What I Learned:**  
How to navigate a web page and locate specific data. In real systems, accessing account data without permission is illegal.

---

### Task 3: Basic Web Weakness Testing

**Task:**  
Test a provided lab URL for common web weaknesses in a safe environment.

**Method:**  
1. Accessed the authorized test URL.  
2. Submitted test input through the form field.  
3. Observed the application response indicating a simulated compromise.

**Result:**  
The lab flagged the submission and displayed a success message: "Bank is hacked" — this is a simulated result for learning.

**What I Learned:**  
Why input validation and access controls are important. Never test systems you don’t own or don’t have written permission for.

---

### Key Takeaways
- Always get written permission before testing any system.
- Document findings to help improve security, not to cause harm.
- Use labs like TryHackMe, HackTheBox, or internal CTFs for practice.
